## 第一轮对话：QQ 通道配置指导

### 用户提问
> 本项目是一个类似openclaw的轻量级AI助手框架，我已经通过项目.venv环境部署完成，但是尚未建立channel与本项目连接，请帮我分析channels/qq.py代码，指导我如何完成本项目与qq实现通道连接。

### 回复内容

## 📋 QQ 通道代码分析

### 1. 架构概览

`channels/qq.py` 实现了基于 **QQ 官方 botpy SDK** 的通道连接，支持以下功能：

- **WebSocket 长连接**：实时接收消息
- **C2C 私聊消息**：支持单聊（`on_c2c_message_create`）
- **私信消息**：支持频道私信（`on_direct_message_create`）
- **自动重连**：断线后 5 秒自动重连

### 2. 配置结构（schema.py）

```python
class QQConfig(Base):
    """QQ channel configuration using botpy SDK."""
    enabled: bool = False
    app_id: str = ""      # 机器人 ID (AppID) from q.qq.com
    secret: str = ""      # 机器人密钥 (AppSecret) from q.qq.com
    allow_from: list[str] = Field(default_factory=list)  # 允许的用户 openid
```

### 3. 通道注册（manager.py）

QQ 通道已在 `ChannelManager` 中注册，当 `config.channels.qq.enabled = true` 时会自动初始化。

## 🔧 配置步骤

### 步骤 1：安装依赖

确保已安装 `qq-botpy` SDK：

```bash
# 在 .venv 环境中执行
pip install qq-botpy
```

### 步骤 2：获取 QQ 机器人凭证

1. 访问 [QQ 开放平台](https://q.qq.com)
2. 注册并创建机器人应用
3. 获取：
   - **AppID**（`app_id`）
   - **AppSecret**（`secret`）

### 步骤 3：配置 nanobot

创建或编辑配置文件（如 `config.yaml` 或环境变量）：

```yaml
channels:
  qq:
    enabled: true
    appId: "你的机器人AppID"      # 或 app_id
    secret: "你的机器人密钥"       # 或 secret
    allowFrom: ["*"]              # 允许所有用户，或指定用户 openid 列表
```

**或使用环境变量**：

```bash
set NANOBOT_CHANNELS__QQ__ENABLED=true
set NANOBOT_CHANNELS__QQ__APP_ID=你的AppID
set NANOBOT_CHANNELS__QQ__SECRET=你的密钥
set NANOBOT_CHANNELS__QQ__ALLOW_FROM=["*"]
```

### 步骤 4：启动 nanobot

```bash
python -m nanobot
```

## ⚠️ 重要注意事项

### 1. 权限控制
- `allow_from: []`（空列表）= **拒绝所有用户**
- `allow_from: ["*"]` = **允许所有用户**
- 建议生产环境指定具体用户 openid

### 2. 消息去重
代码中已实现消息 ID 去重（`self._processed_ids`），防止重复处理

### 3. 支持的交互类型
- ✅ C2C 私聊消息（单聊）
- ✅ 频道私信
- ❌ 群聊（需要额外申请权限）

### 4. 日志查看
启动后会显示：
```
QQ bot started (C2C private message)
QQ bot ready: {机器人名称}
```

## ⚠️ 识别的安全风险

### 🔴 高风险

1. **明文存储 API 密钥**
   - 配置文件 `~/.nanobot/config.json` 以明文存储所有密钥
   - 位置：loader.py

2. **缺少速率限制**
   - 用户可以无限发送消息
   - 可能导致 API 费用激增或 DDoS

3. **日志可能泄露敏感信息**
   - 日志可能包含用户消息、命令输出等

### 🟡 中风险

4. **会话管理缺失**
   - 没有会话过期机制
   - 长期运行的会话可能积累过多上下文

5. **审计日志不足**
   - 缺少完整的安全事件追踪

6. **命令过滤有限**
   - shell.py 只阻止明显的危险模式
   - 可能存在绕过方式

### 🟢 低风险

7. **配置文件权限未强制**
   - 没有自动设置 `chmod 600` 的机制

8. **环境变量支持不完整**
   - 虽然支持环境变量，但文档不够清晰

## 🛡️ 安全强化实施方案

### 1. 配置文件安全强化

#### 创建安全配置文件模板

```json
{
  "channels": {
    "qq": {
      "enabled": true,
      "appId": "YOUR_QQ_APP_ID",
      "secret": "YOUR_QQ_SECRET",
      "allowFrom": ["*"]
    },
    "telegram": {
      "enabled": false,
      "token": "YOUR_TELEGRAM_TOKEN",
      "allowFrom": ["USER_ID_1", "USER_ID_2"]
    }
  },
  "tools": {
    "restrictToWorkspace": true,
    "exec": {
      "timeout": 60,
      "pathAppend": ""
    }
  },
  "providers": {
    "custom": {
      "apiKey": "YOUR_API_KEY",
      "apiBase": "https://api.example.com"
    }
  }
}
```

#### 设置文件权限（PowerShell）

```powershell
# 设置配置文件权限（仅当前用户可读写）
$acl = Get-Acl "$env:USERPROFILE\.nanobot\config.json"
$accessRule = New-Object System.Security.AccessControl.FileSystemAccessRule(
    $env:USERNAME,
    "FullControl",
    "Allow"
)
$acl.SetAccessRule($accessRule)
# 禁用继承
$acl.SetAccessRuleProtection($true, $false)
Set-Acl "$env:USERPROFILE\.nanobot\config.json" $acl
```

### 2. 使用环境变量存储敏感信息

```powershell
# 设置环境变量（仅当前会话）
```

### 5. 创建安全启动脚本

创建 `start-secure.ps1`：

```powershell
# 安全启动脚本
$ErrorActionPreference = "Stop"

# 检查配置文件权限
$configPath = "$env:USERPROFILE\.nanobot\config.json"
if (Test-Path $configPath) {
    $acl = Get-Acl $configPath
    $access = $acl.Access | Where-Object { $_.IdentityReference -eq $env:USERNAME }
    if ($access.FileSystemRights -notlike "*FullControl*") {
        Write-Warning "配置文件权限不安全，正在修复..."
        # 设置权限代码（同上）
    }
}

# 检查工作区限制
$config = Get-Content $configPath | ConvertFrom-Json
if (-not $config.tools.restrictToWorkspace) {
    Write-Warning "建议启用 restrictToWorkspace 以增强安全性"
}

# 启动 nanobot
Write-Host "启动 nanobot（安全模式）..." -ForegroundColor Green
python -m nanobot
```

### 6. 增强命令过滤（可选）

编辑 shell.py，添加更多危险模式：

```python
self.deny_patterns = deny_patterns or [
    r"\brm\s+-[rf]{1,2}\b",          # rm -r, rm -rf, rm -fr
    r"\bdel\s+/[fq]\b",              # del /f, del /q
    r"\brmdir\s+/s\b",               # rmdir /s
    r"(?:^|[;&|]\s*)format\b",       # format
    r"\b(mkfs|diskpart)\b",          # disk operations
    r"\bdd\s+if=",                   # dd
    r">\s*/dev/sd",                  # write to disk
    r"\b(shutdown|reboot|poweroff)\b",  # system power
    r":\(\)\s*\{.*\};\s*:",          # fork bomb
    # 新增：Windows 特定危险命令
    r"\bcd\s+/[dD]",                 # cd /d (可能改变驱动器)

    r"\breg\s+(add|delete|import|export)\b",  # registry operations
```

### 8. 定期安全检查脚本

创建 `security-check.ps1`：

```powershell
Write-Host "=== Nanobot 安全检查 ===" -ForegroundColor Cyan

# 1. 检查配置文件权限
$configPath = "$env:USERPROFILE\.nanobot\config.json"
if (Test-Path $configPath) {
    $acl = Get-Acl $configPath
    $hasOtherAccess = $acl.Access | Where-Object { 
        $_.IdentityReference -notmatch $env:USERNAME -and 
        $_.IdentityReference -notmatch "SYSTEM" -and
        $_.IdentityReference -notmatch "BUILTIN\\Administrators"
    }
    if ($hasOtherAccess) {
        Write-Host "❌ 配置文件权限不安全：存在其他用户访问权限" -ForegroundColor Red
    } else {
        Write-Host "✅ 配置文件权限安全" -ForegroundColor Green
    }
} else {
    Write-Host "⚠️  配置文件不存在" -ForegroundColor Yellow
}

# 2. 检查工作区限制
if (Test-Path $configPath) {
    $config = Get-Content $configPath | ConvertFrom-Json
    if ($config.tools.restrictToWorkspace) {
        Write-Host "✅ 工作区限制已启用" -ForegroundColor Green
    } else {
        Write-Host "⚠️  工作区限制未启用（建议启用）" -ForegroundColor Yellow
    }
}

# 3. 检查依赖更新
Write-Host "`n检查依赖安全更新..." -ForegroundColor Cyan
pip list --outdated --format=json | ConvertFrom-Json | Where-Object { 
    $_.name -match "(litellm|httpx|botpy)" 
} | ForEach-Object {
    Write-Host "⚠️  $($_.name): $($_.version) -> $($_.latest_version)" -ForegroundColor Yellow
}

# 4. 检查日志文件大小
$logPath = "$env:USERPROFILE\.nanobot\logs\nanobot.log"
if (Test-Path $logPath) {
    $logSize = (Get-Item $logPath).Length / 1MB
    if ($logSize -gt 10) {
        Write-Host "⚠️  日志文件过大：$([math]::Round($logSize, 2)) MB" -ForegroundColor Yellow
    } else {
        Write-Host "✅ 日志文件大小正常" -ForegroundColor Green
    }
}

Write-Host "`n=== 检查完成 ===" -ForegroundColor Cyan
```
